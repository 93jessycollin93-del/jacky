# Skills System

Each skill lives under `skills/<name>/` and contains:

- `SKILL.md` with YAML frontmatter and Markdown instructions
- `scripts/` for optional automation
- `references/` for checklists and source material
- `assets/` for templates and examples

## Skill Design Rules

- Keep skills focused on one repeatable workflow.
- Define clear inputs and outputs.
- Include verification and safety notes.
- Prefer reusable templates over long agent prompts.
